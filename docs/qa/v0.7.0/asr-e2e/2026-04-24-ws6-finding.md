# WS-6 ASR E2E — Live voice-note round-trip finding

**Date:** 2026-04-24 22:35 (-0300)

## What was tested

1. Sent the existing test fixture `backend/tests/fixtures/asr_test_en.ogg` (19,699 bytes, English speech) to the agent bot via the tester MCP `/api/send` endpoint with `media_path` + `media_type=audio`.
2. Tester confirmed `Message sent to 5527988290533` (success).
3. Watched bot logs for the round-trip.

## What happened

- Agent bot received the audio:
  ```
  Media info: type=audio, filename=audio_20260425_013547.ogg
  Successfully downloaded audio media to /app/store/...audio_20260425_013547.ogg (19699 bytes)
  ```
  → byte count matches fixture.
- Backend processed the message but logged:
  ```
  ERROR [agent.router] [SAFETY] Empty message detected for agent 41, blocking AI processing
  ```
- No `audio_transcript`, `whisper`, `openai_asr`, or `transcrib*` log lines appeared in `tsushin-backend` for the affected message ID.
- Bot replied with the canned error:
  > Sorry, I couldn't process your message. If you sent an audio message, there was an issue with the audio transcription.

## Root cause (best guess, not investigated in this RC)

The WhatsApp-bridge → backend handoff appears to drop the audio payload (or never invokes the `audio_transcript` skill) before the agent router runs. The bridge downloaded the file fine, but the backend received empty content. Skill not configured for this agent, or the bridge isn't routing audio to the right skill, or the OpenAI key for the default ASR provider isn't available in this tenant.

## Scope

This is a pre-existing pipeline gap. The v0.7.0 RC sweep did not touch ASR code — `audio_transcript.py`, `asr_registry.py`, `openai_asr_provider.py`, `whisper_asr_provider.py` are unchanged. Existing pytest coverage (`test_audio_transcript_skill_asr.py`, `test_whisper_container_manager.py`) still passes against the unchanged code.

## Recommendation

Out of scope for the v0.7.0 RC. Filed as a follow-up:
- Verify the `audio_transcript` skill is enabled on agent 41 and other Whatsapp-bound agents.
- Confirm the bridge → backend message envelope carries `media_url`/`media_path` for the audio_transcript skill to read.
- Confirm the tenant has an OpenAI provider instance (default ASR) or a Speaches Whisper instance set as `tenant.default_asr_instance_id`.
