# v0.7.0 FK Cascade Decisions

Tenant isolation remains mandatory, but audit-style rows should survive tenant
cleanup unless an explicit data-retention task removes them.

| Table | FK | Delete behavior | Rationale |
| --- | --- | --- | --- |
| `channel_event_dedupe` | `tenant_id -> tenant.id` | `RESTRICT` | Dedupe outcomes are security/audit evidence and should not disappear automatically. |
| `channel_event_dedupe` | future instance references | no FK in `0045` | Instance tables vary by trigger type; later tracks validate ownership before writes. |
| `wake_event` | `tenant_id -> tenant.id` | `RESTRICT` | Wake events are audit surfaces for continuous agents. |
| `continuous_run` | `tenant_id -> tenant.id` | `RESTRICT` | Run history is retained for audit and incident review. |
| `continuous_agent` | `agent_id -> agent.id` | `CASCADE` | Continuous-agent runtime config is owned by its base agent. |
| `continuous_agent` | `delivery_policy_id -> delivery_policy.id` | `SET NULL` | Policy removal should not delete the agent wrapper. |
| `continuous_agent` | `budget_policy_id -> budget_policy.id` | `SET NULL` | Policy removal should not delete the agent wrapper. |
| `continuous_agent` | `approval_policy_id -> sentinel_profile.id` | `SET NULL` | Sentinel profiles can be replaced without deleting runtime config. |
| `continuous_subscription` | `tenant_id -> tenant.id` | `CASCADE` | Subscription config is tenant-owned, not audit evidence. |
| `continuous_subscription` | `continuous_agent_id -> continuous_agent.id` | `CASCADE` | Subscriptions are owned by the continuous-agent definition. |
| `continuous_subscription` | `delivery_policy_id -> delivery_policy.id` | `SET NULL` | Subscription can fall back to agent/default policy. |
| `wake_event` | `continuous_agent_id -> continuous_agent.id` | `SET NULL` | Wake events are audit surfaces and survive agent config deletion. |
| `wake_event` | `continuous_subscription_id -> continuous_subscription.id` | `SET NULL` | Wake events retain the original channel metadata if a subscription is removed. |
| `continuous_run` | `continuous_agent_id -> continuous_agent.id` | `CASCADE` | Run records remain tenant-restricted; deleting the owning continuous agent removes its internal run rows after retention review. |
| `delivery_policy` / `budget_policy` | `tenant_id -> tenant.id` | `CASCADE` | Policies are tenant configuration, not immutable audit evidence. |
| `channel_event_rule` | `tenant_id -> tenant.id` | `CASCADE` | Routing rules are mutable tenant/channel configuration. |
| `channel_event_rule` | `agent_id -> agent.id` | `CASCADE` | Rules are invalid without the target agent. |
| `channel_event_rule` | `channel_instance_id` | no FK in `0050` | Instance tables vary by channel type; API/service code validates ownership before writes. |

Phase 0 creates only `channel_event_dedupe`; later tracks must update this
document before adding new FK-bearing tables.
