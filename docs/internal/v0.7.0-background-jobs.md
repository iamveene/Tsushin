# v0.7.0 Background Job Durability

v0.7.0 uses the existing PostgreSQL `message_queue` table rather than adding
Temporal, pg-boss, or another worker dependency.

Phase 0 contract:

- `message_queue.message_type` is the durable discriminator.
- Supported values are `inbound_message`, `trigger_event`, and `continuous_task`.
- `agent_id` and `sender_key` stay non-null in Phase 0. Trigger and continuous
  rows must already have resolved agent context until later wake-event models land.
- Existing rows default to `inbound_message`.
- Queue dispatch first branches on `message_type`, then uses the existing channel
  handlers for current inbound traffic.
- Unsupported Phase 0 `trigger_event` or `continuous_task` combinations fail
  explicitly instead of being silently completed.
- Jira and Email finalization keep managed WhatsApp notifications inside the
  scheduler poll path for v0.7.0. `continuous_run` rows are still written for
  audit, but the generic `continuous_task` queue consumer remains intentionally
  out of scope.

Postgres is the only migration target for v0.7.0. SQLite compatibility is best-effort for unit tests only.
