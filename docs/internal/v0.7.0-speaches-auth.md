# v0.7.0 Speaches Auth Contract

Whisper/Speaches ASR containers will use per-instance Basic Auth.

Contract:

- Generate a cryptographically random token when provisioning the ASR instance.
- Store only the encrypted token in the instance row using the same tenant-scoped encryption conventions as other Hub secrets.
- Inject `SPEACHES_API_KEY` into the container environment.
- The ASR provider must call Speaches with `Authorization: Basic <base64("tsushin:<token>")>`.
- Health checks must use the same auth path as transcription calls.
- Token rotation is a later v0.7.x operation unless an instance is reprovisioned.

Port allocation:

- Whisper/Speaches uses `PORT_RANGES["whisper"] == (6400, 6499)`.
- SearXNG remains on `6500-6599`.

Warm-up:

- After the container reports running, Phase 5 sends a one-second silent clip to force model load before marking the instance healthy.
