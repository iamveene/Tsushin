Rules:

- Do not create `track/*` worktrees until Phase 0 is merged and validated.
- Do not reuse or renumber `0045`; it is the serial Phase 0 foundation.
- Track F must not modify `continuous_run`; Track A creates that table in `0047`.
- Every migration must include downgrade handling and idempotency checks where partial-state recovery is plausible.
- The current release chain is linear and intentionally out of numeric order:
  `0044 -> 0045 -> 0046 -> 0048 -> 0051 -> 0059 -> 0047 -> 0050 -> 0049 -> 0057 -> 0058`.
- The Track B dispatch foundation is a no-migration slice. The adapter breadth migrations chain as `0058 -> 0052 -> 0053 -> 0054`, with `0052.down_revision = "0058"`. Phase 3 Sentinel hardening then chains `0054 -> 0055`, Jira finalization chains `0055 -> 0060`, and Email trigger parity chains `0060 -> 0061`.
